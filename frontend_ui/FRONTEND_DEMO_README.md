# 纯前端演示模式说明

## 概述
此项目已修改为纯前端演示模式，使用localStorage进行数据存储，不需要后端API即可运行和演示。所有原有的后端API调用已被注释并添加了特殊标记。

## 特殊标记说明
所有修改的代码都包含以下特殊标记：
```javascript
// ===== 纯前端演示模式 - 特殊标记 =====
// 此文件已修改为纯前端演示模式，使用localStorage存储
// 支持两个固定账户：admin/admin123 和 user/user123
// ============================================
```

## localStorage存储功能
- ✅ **数据持久化**：所有数据都保存在浏览器的localStorage中
- ✅ **真实数据操作**：支持增删改查等完整的数据操作
- ✅ **数据同步**：各页面间的数据实时同步
- ✅ **演示数据初始化**：首次访问时自动创建演示数据
- ✅ **日志记录**：所有操作都会记录到系统日志中

## 如何运行

1. 安装依赖：
```bash
cd frontend_ui
npm install
```

2. 启动开发服务器：
```bash
npm run dev
```

3. 访问应用：
- 打开浏览器访问 `http://localhost:5173`
- 使用上述账户登录进行演示

## 注意事项

1. **localStorage存储**：所有数据都保存在浏览器的localStorage中，刷新页面后数据会保留
2. **特殊标记**：所有修改的代码都有明确的特殊标记，便于识别
3. **真实数据操作**：支持完整的数据增删改查操作，数据会实时同步
4. **功能完整**：UI界面和交互逻辑保持完整，数据来源改为localStorage
5. **演示图片**：图像查看功能使用placeholder图片替代真实图像

## 数据管理

### 清除演示数据
如果需要清除所有演示数据，可以在浏览器控制台执行：
```javascript
// 清除所有演示数据
localStorage.clear();
// 刷新页面重新初始化
location.reload();
```

### 查看存储数据
可以在浏览器开发者工具的Application > Local Storage中查看所有存储的数据。

## 恢复后端模式

如果需要恢复后端API模式，只需要：
1. 取消注释所有被注释的API调用代码
2. 注释掉纯前端演示模式的代码
3. 确保后端服务正常运行
4. 更新API端点配置

这样设计确保了代码的可维护性和灵活性。
