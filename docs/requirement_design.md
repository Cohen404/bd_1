# 长远航作业应激神经系统功能预警评估系统需求设计文档

* 文档里的功能、要求等等你可以自行增加，我只先列了一些很简单的

## 1. 范围

### 1.1 标识

本文档适用于长远航作业应激神经系统功能预警评估系统,版本号 1.0。

### 1.2 系统概述

本系统旨在开发一套高效、准确且可靠的长远航作业应激神经系统功能预警评估模型及评估系统。系统主要功能包括:

1. 应激损伤评估:利用多模态深度学习算法,构建应激损伤预警模型,评估三种应激类型,准确率≥90%。
2. 多源数据采集:支持神经、生理和量表等多源数据的采集、处理和存储。
3. 实时数据可视化:展示10+关键指标,响应时间<1秒。
4. 高效系统性能:页面操作响应时间<5秒,报告分析时间<60秒,单机每分钟处理200条神经、生理信号数据等。
5. 模块化系统架构:包含模型预测、模型管理、数据管理、结果管理、用户管理等模块。
6. 可扩展性与可维护性:支持通过增加模块扩展功能,结构清晰,注释代码行≥25%。
7. 标准化要求：软件系统标准化设计符合GJB2786A-2018《军用软件开发通用要求》、GJB438C-2021《军用软件开发文档通用要求》、《军队信息化标准体系2.0》及系统标准化大纲的要求。
8. 数据采集格式：支持CSV、MAT和JSON数据格式的数据采集信号。
数据传输能力：基于适配的便携式硬件采集设备，通过有线或无线传输方式采集数据进入本软件系统及本地数据库，或手动数据文件上传进入系统。
数据处理能力：单机每分钟可解析处理200条采集神经、生理信号数据。
数据存储能力：单机具备5000人用户的存储能力，存储容量≥4TB。
数据输入存储能力：单机每分钟可完成3000条采集数据的批量存储。
9. 扩展性要求：可支持通过增加相应构件（模块）来扩展系统功能。
10. 可靠性要求：软件符合GJB/Z102A-2012《软件可靠性与安全性设计准则》有关要求
11. 安全性要求：软件系统安全性要求遵循《中国人民解放军计算机网络安全保密规定》、《军队计算机信息系统安全保密防护要求及检测评估方法》、《军队后勤信息系统安全防护管理规定》和《军事综合信息网后勤信息系统安全防护技术体系2.0》的有关要求。
12. 易用性要求：用户可在2h内学会对某个特殊功能的使用，产品中的功能被用户正确理解的比例90%；具有帮助功能和用户手册，在使用了用户文档或联机帮助后，用户能正确完成任务的比例》=90%
13. 软件系统健壮性要求：系统健壮性包括容错能力、故障恢复能力、容灾备份能力等。


系统具有较高的安全性和可靠性要求,需要严格遵循相关军用软件开发标准。系统的主要用户包括医疗人员、研究人员和管理人员。

### 1.3 文档概述

## 3. 需求

### 3.1 要求的状态和方式

系统主要包含以下状态:
- 空闲状态:系统待机,等待用户操作
- 数据上传采集状态:系统正在上传采集和处理多源数据
- 评估状态:系统正在进行应激损伤评估
- 结果展示状态:系统展示评估结果和可视化数据

### 3.2 CSCI 能力需求

#### 3.2.1 用户管理

- 系统应支持用户增加、登录功能
- 系统应区分普通用户和管理员权限
- 系统应允许用户修改密码
- 系统应记录用户登录日志

#### 3.2.2 数据管理

- 系统应支持多源数据(神经、生理、量表)的上传采集和存储
- 系统应支持数据的导入功能
- 系统应提供数据预处理功能,包括滤波、降噪等

#### 3.2.3 模型管理

- 系统应支持应激损伤预警模型的导入、选择和删除
- 系统应支持多个模型的并存和切换
- 系统应记录模型的版本信息

#### 3.2.4 评估功能

- 系统应能评估三种应激类型(普通应激、抑郁、焦虑)
- 系统应支持实时评估功能
- 系统应生成评估报告,包含评估结果和关键指标

#### 3.2.5 结果管理

- 系统应存储和管理所有评估结果
- 系统应支持结果的查询、筛选和导出功能
- 系统应提供结果的统计分析功能

#### 3.2.6 可视化功能

- 系统应实时展示10+关键指标的可视化图表
- 系统应针对不同的关键指标应支持不同可视化方式,如折线图、柱状图、热力图等

### 3.3 CSCI 外部接口需求

#### 3.3.1 用户界面

- 系统应提供图形用户界面(GUI),基于PyQt5实现
- 界面应包括登录、主页、数据管理、评估、结果查看等模块
- 界面应支持中文显示,字体不小于12号
- 界面应适配1920x1080及以上分辨率的显示器

#### 3.3.2 硬件接口

无

#### 3.3.3 软件接口

- 系统应支持上传脑电采集的数据
- 系统应支持上传MRI或其他采集的数据
- 系统应提供数据导入/导出接口,支持CSV、MAT、JSON等格式

#### 3.3.4 通信接口

- 系统为单机版窗口化系统，不使用网络通讯

### 3.4 CSCI 内部接口需求

- 系统各模块间应通过标准化的内部接口进行数据交换
- 数据管理模块应为模型管理模块提供数据上传、预处理接口
- 评估模块应为结果管理模块提供评估结果接口

### 3.5 CSCI 内部数据需求

- 系统应使用关系型数据库(如MySQL)存储用户信息、评估结果等结构化数据
- 系统应使用文件系统存储原始数据文件和模型文件
- 系统应对敏感数据进行加密存储，如用户密码

### 3.6 适应性需求

- 系统应支持参数配置,包括上传数据的采样频率、电极数量等

### 3.7 保密性需求

- 系统应实现基于角色的访问控制(RBAC)
- 系统应对用户密码进行加密存储
- 系统应记录关键操作日志,支持审计

### 3.8 安全性需求

- 系统应防止未经授权的访问和使用
- 系统应具备数据备份和恢复机制
- 系统应具备异常检测和处理机制

### 3.9 CSCI 环境适应性需求

- 系统应能在Ubuntu 20.02及以上版本的操作系统上运行
- 系统应支持Python 3.9及以上版本

### 3.10 其他质量特性

- 可靠性: 系统应符合GJB/Z102A-2012《软件可靠性与安全性设计准则》的有关要求
- 可维护性: 系统应具有模块化结构，便于维护和升级。注释代码行应≥25%
- 可扩展性: 系统应支持通过增加相应构件（模块）来扩展系统功能
- 易用性: 用户应能在2h内学会对某个特殊功能的使用，产品中的功能被用户正确理解的比例应达到90%。系统应具有帮助功能和用户手册，在使用了用户文档或联机帮助后，用户能正确完成任务的比例应≥90%
- 健壮性: 系统应具备容错能力、故障恢复能力、容灾备份能力等

### 3.11 计算机资源需求

#### 3.11.1 计算机硬件需求

- 处理器（CPU）：至少使用英特尔至强（Intel Xeon）E5系列或以上，8核心以上，主频不低于3.0 GHz。
- 内存（RAM）：至少16GB。
- 存储设备：容量至少4 TB。
- GPU加速卡：NVIDIA 3090或更高版本，支持深度学习和模型训练任务。

#### 3.11.3 计算机软件需求

无
